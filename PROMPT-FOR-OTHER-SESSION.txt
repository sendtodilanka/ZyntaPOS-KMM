│ PRIORITY 1 — Backend DB Queries (stub → real implementation)               │
└─────────────────────────────────────────────────────────────────────────────┘
All three Ktor services have TODO stubs where real Exposed ORM queries belong.
Implement them:

  backend/api/service/UserService.kt
    • authenticate(licenseKey, deviceId, username, password): User?
      → Query `users` table by email/username, verify bcrypt password hash,
        check `stores` table for licenseKey match, return User or null
    • refreshTokens(refreshToken, config): LoginResponse?
      → Verify JWT refresh token (RS256), load user from DB, issue new tokens

  backend/api/service/SyncService.kt
    • push(operations: List<SyncOperation>, storeId: String): SyncPushResponse
      → Insert rows into `sync_queue` table, resolve conflicts via last-write-wins,
        update `version_vectors` table
    • pull(storeId: String, since: Instant): SyncPullResponse
      → Query `sync_queue` WHERE store_id = storeId AND updated_at > since
        ORDER BY updated_at ASC, return paginated list

  backend/api/service/ProductService.kt
    • getProducts(storeId, updatedSince, page, pageSize): ProductListResponse
      → Query `products` table with optional updatedSince filter, paginate

  backend/license/service/LicenseService.kt
    • activate(request): ActivateResponse?
      → Query `licenses` table by key, check seat count vs device_registrations,
        insert new device_registration row, return activation result
    • heartbeat(request): HeartbeatResponse?
      → Update `device_registrations.last_seen_at`, return license status
    • getStatus(key): LicenseStatusResponse?
      → Query `licenses` JOIN `device_registrations`, return counts

  All DB queries use Jetbrains Exposed DSL (already imported in DatabaseFactory).
  Schema is already created by Flyway migrations in:
    backend/api/src/main/resources/db/migration/V1__initial_schema.sql
    backend/api/src/main/resources/db/migration/V2__licenses_table.sql
    backend/license/src/main/resources/db/migration/V1__license_schema.sql
